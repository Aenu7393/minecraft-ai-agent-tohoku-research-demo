# Minecraft AI Agent Research Demo

## 概要

このリポジトリは、Minecraft内で動作する自律AIエージェントに関する研究プロジェクトの公開用デモ版です。

本プロジェクトの目的は、エージェントが現在のゲーム状況を理解し、高レベルな目標を実行可能なサブゴールへ分解し、行動を実行し、経験を記憶し、失敗から回復できる仕組みを構築することです。

本研究では、LLMを単に行動計画の生成に使うだけでなく、人間の記憶システムに着想を得た、記憶ベースのエージェントアーキテクチャの設計に重点を置いています。

## 研究の背景・目的

大規模言語モデルは高レベルな計画を生成できますが、その計画を実際の環境状況に結びつけることは得意ではありません。

例えば、LLMが「木を集める → 板材を作る → 棒を作る → ツルハシを作る」という計画を生成できたとしても、実際に行動するためには、現在の所持アイテム、周囲のブロック、サブゴールの実行可能性、失敗理由、過去の経験の再利用などを判断する必要があります。

この問題に対処するため、本プロジェクトでは、LLMによる計画生成に加えて、状態表現、記憶、失敗回復を組み合わせたエージェントの構築を目指しています。

## 主要なアイデア

本プロジェクトでは、人間の記憶の種類や機能を参考にして、エージェントの記憶システムを設計しています。

エージェントは、現在の状態や直近の行動を一時的に保持するだけでなく、重要な経験を整理し、長期的に再利用できる知識として保存することを目指しています。

記憶システムは、主に以下のように構成しています。

```text
Memory System
├── Working Memory
│   └── 現在の目標・サブゴール・状態・直近の実行状況
│
├── Intermediate Memory
│   └── 直近の経験を一時的に保持し、重要な情報を長期記憶へ移行するための領域
│
└── Long-Term Memory
    ├── Declarative Memory
    │   ├── Episodic Memory
    │   │   └── 過去の経験、五感に相当する状態情報、失敗、反省など
    │   └── Semantic Memory
    │       └── クラフトレシピ、アイテム関係、タスク依存関係など
    │
    └── Non-Declarative Memory
        ├── Associative Learning
        │   ├── Classical Conditioning
        │   └── Operant Conditioning
        ├── Non-Associative Learning
        ├── Priming
        ├── Procedural Memory
        └── Emotional Conditioning
```

## 中期記憶から長期記憶への移行

本プロジェクトでは、すべての経験をそのまま長期記憶へ保存するのではなく、一度中期記憶に保持し、重要なものだけを長期記憶へ移行する仕組みを検討しています。

例えば、以下のような情報は長期記憶へ移行する候補になります。

* 同じ失敗が繰り返された経験
* 有効だった修復方法
* 他のタスクにも再利用できる知識
* タスク達成に重要だった視覚情報
* 危険や成功と強く結びついた状況
* 特定の行動と結果の関係

この仕組みにより、エージェントは単に経験を蓄積するだけでなく、重要な経験を選別して知識化することを目指しています。

```text
現在の状態・経験
   ↓
Working Memory
   ↓
Intermediate Memory
   ↓
重要度・新規性・再利用性・失敗頻度を評価
   ↓
Long-Term Memory
```

## 現在のアプローチ

これまでのバージョンでは、Mineflayerを用いてMinecraftから構造化された状態情報を取得していました。

例えば、インベントリ、位置情報、周囲のブロック、体力、現在のタスク、実行結果などを取得し、それらを計画、実行、失敗回復のための状態ベクトルとして利用していました。

しかし、Mineflayerによる状態表現は記号的な情報が中心であり、ゲーム画面全体の視覚的な状況を十分に表現することは難しいです。

そのため現在は、Minecraftの画面画像を状態表現の一部として利用する方向に拡張しています。

## 画像ベースの状態表現

現在取り組んでいる方向性は、Minecraftのスクリーンショットから視覚情報を取得し、それを状態ベクトルとして利用することです。

この画像ベースの状態ベクトルは、以下のような用途に利用することを想定しています。

* 現在の環境をより詳細に理解する
* 軽い判断を支援する
* 環境の変化を検出する
* サブゴールの実行を改善する
* 失敗時の反省を支援する
* 現在の計画がうまくいかない場合にサブゴールを修正する

テキストベースや記号ベースの状態情報だけに依存するのではなく、視覚情報を取り入れることで、エージェントがゲーム内の状況をより正確に理解できるようにすることを目指しています。

## 非連合学習の応用

本研究では、非連合学習の一つである **馴化** の考え方を、画像ベースの状態理解に応用することを検討しています。

馴化とは、重要でない刺激が繰り返し与えられたときに、その刺激への反応や注意が徐々に小さくなる現象です。

Minecraftエージェントにおいては、繰り返し観測される重要度の低い視覚情報への注意を下げ、新規性が高い部分、不確実な部分、タスクに関係する部分へより注目する仕組みとして応用したいと考えています。

これにより、不要な観測コストを減らしながら、失敗時の反省やサブゴール修正に必要な情報へ集中できるようにすることを目指しています。

## アーキテクチャ

```text
ユーザーの目標
   ↓
LLM Planner
   ↓
Subgoal Queue
   ↓
Meta Controller
   ↓
Executor
   ↓
Minecraft Agent
```

エージェントは、計画、実行、失敗回復の各段階で、記憶と状態表現を利用します。

```text
State Observer
   ├── Mineflayerから取得した記号的状態
   └── Minecraftスクリーンショットから取得した視覚的状態

Memory System
   ├── Working Memory
   ├── Intermediate Memory
   └── Long-Term Memory
```

## タスク例

入力目標：

```text
石のツルハシを作る
```

サブゴール列の例：

```text
find_tree
chop_tree
craft_planks
craft_crafting_table
craft_sticks
craft_wooden_pickaxe
mine_cobblestone
craft_stone_pickaxe
```

もしこれらのサブゴールの途中で失敗した場合、エージェントは現在の状態、過去の失敗事例、視覚的な状態表現を確認し、修復戦略を生成または再利用します。

## 主な機能

* LLMによる目標のサブゴール分解
* サブゴール単位のタスク実行
* Mineflayerを用いた記号的状態観測
* Minecraftスクリーンショットを用いた視覚的状態表現
* 作業記憶・中期記憶・長期記憶を用いた記憶アーキテクチャ
* エピソード記憶による経験・失敗・反省の保存
* 意味記憶によるクラフト知識やタスク依存関係の保存
* 手続き記憶による再利用可能なスキルの保存
* 非連合学習における馴化を用いた視覚的注意の調整
* メタコントローラによる次の行動モードの決定
* 失敗時のサブゴール修正と修復戦略の再利用
